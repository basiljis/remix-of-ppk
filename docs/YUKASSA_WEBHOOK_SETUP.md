# Настройка Webhook от ЮКассы

## Шаги настройки

### 1. Получите URL webhook
URL вашего webhook endpoint:
```
https://oxyjmeslnmhewlpgzlmf.supabase.co/functions/v1/yukassa-webhook
```

### 2. Настройка в личном кабинете ЮКассы

1. Войдите в [личный кабинет ЮКассы](https://yookassa.ru/)
2. Перейдите в раздел "Настройки" → "Уведомления"
3. Включите уведомления о платежах
4. Добавьте URL webhook (указанный выше)
5. Выберите события для отправки:
   - ✅ `payment.succeeded` - платеж успешен
   - ✅ `payment.canceled` - платеж отменен (опционально)
   - ✅ `refund.succeeded` - возврат выполнен (опционально)

### 3. Проверка подписи (опционально)

ЮКасса может отправлять подпись в заголовке `x-yookassa-signature` для проверки подлинности уведомлений.

Webhook автоматически проверяет подпись, если она присутствует в запросе.

### 4. Тестирование

После настройки:

1. Создайте тестовый платеж через форму подписки
2. Завершите оплату в тестовом режиме
3. Проверьте логи webhook: https://supabase.com/dashboard/project/oxyjmeslnmhewlpgzlmf/functions/yukassa-webhook/logs
4. Убедитесь, что подписка активировалась автоматически

### 5. Логирование

Все события webhook логируются в таблицу `payment_logs`:
- ID платежа
- Тип события
- Статус
- Сумма
- Полные данные в формате JSON
- Флаг обработки

## Обработка ошибок

Если webhook не срабатывает:

1. Проверьте логи функции в Supabase Dashboard
2. Убедитесь, что URL webhook правильно настроен в ЮКассе
3. Проверьте, что edge function развернута
4. Проверьте таблицу `payment_logs` на наличие записей

## Безопасность

- ✅ Проверка подписи webhook (если включена)
- ✅ Идемпотентность обработки (один платеж не обработается дважды)
- ✅ Логирование всех событий
- ✅ Row-Level Security на таблицах

## Типы событий

| Событие | Описание | Действие |
|---------|----------|----------|
| `payment.succeeded` | Платеж успешно завершен | Активация подписки, отправка email |
| `payment.canceled` | Платеж отменен | Логирование (без дополнительных действий) |
| `payment.pending` | Платеж в процессе | Логирование |
| `refund.succeeded` | Возврат выполнен | Логирование |
